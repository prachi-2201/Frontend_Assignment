Q. Explain Life cycle in Class Component and functional component with Hooks

React, component lifecycle refers to the sequence of phases a component goes through from its initialization to its destruction. The lifecycle methods/hooks allow you to perform certain actions at specific points during this process. Let's delve into the lifecycles for both class components and functional components with hooks:

Class Components:

Lifecycle Methods:

  1. Mounting Phase:
    • constructor(): Initializes the component state and binds event handlers.
    • componentWillMount() (deprecated): Invoked just before rendering for the first time. Avoid using it as it's considered legacy.
    • render(): Returns the JSX representation of the component.
    • componentDidMount(): Executed after the component is rendered for the first time. Useful for initiating network requests or setting up subscriptions.
  2. Updating Phase:
    • componentWillReceiveProps(nextProps) (deprecated): Called when the component is receiving new props.
    • shouldComponentUpdate(nextProps, nextState): Determines whether the component should re-render.
    • componentWillUpdate(nextProps, nextState) (deprecated): Invoked just before rendering when new props or state are received.
    • render(): Renders the updated JSX.
    • componentDidUpdate(prevProps, prevState): Executed after the component is re-rendered due to changes in props or state.
  3. Unmounting Phase:
    • componentWillUnmount(): Invoked just before the component is removed from the DOM. Cleanup tasks like cancelling timers or subscriptions are performed here.

Functional Components with Hooks:

Hooks:

  1. useState: Allows functional components to manage local state.
  2. useEffect: Mimics the combined functionality of various lifecycle methods.
