Q.1 What is Redux?

Redux is a predictable state container for JavaScript apps. It helps manage the state of your application in a centralized store, making state management easier and more predictable, especially in large-scale applications.
It operates with a unidirectional data flow, meaning data flows in one direction within your application, making it easier to understand and debug.

Q.2 What is Redux Thunk used for?

Redux Thunk is a middleware for Redux that allows you to write asynchronous logic that interacts with the Redux store.
It is commonly used to handle asynchronous actions in Redux, such as making API calls, by allowing action creators to return functions instead of plain objects.
This enables you to perform asynchronous operations before dispatching actions, giving you more control over the flow of data in your Redux application.

Q.3 What is Pure Component? When to use Pure Component over Component?

A Pure Component is a component in React that implements the shouldComponentUpdate method with a shallow prop and state comparison.
It ensures that the component only re-renders when its props or state have changed. If the shallow comparison indicates that the props and state are the same, the component won't re-render.
Use Pure Components when you want to optimize performance by reducing unnecessary re-renders. They are particularly useful when dealing with large lists or deeply nested component trees where re-rendering can be costly.

Q.4 What is the second argument that can optionally be passed tosetState and what is its purpose?

The second argument that can optionally be passed to setState in React is a callback function that will be executed once setState has been completed and the component has been re-rendered.
This callback function is often used to perform actions that depend on the updated state or to execute code after the component has been re-rendered.
It's particularly useful for scenarios where you need to perform actions after the state has been updated and reflected in the UI, such as fetching data from an API or manipulating the DOM based on the updated state.
